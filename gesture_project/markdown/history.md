# Jetson Nano & Arduino UNO Q 제스처 기반 객체 추적 프로젝트 가이드

본 문서는 Jetson Nano(개발 환경)에서 학습 및 검증을 완료한 후, 퀄컴 드래곤윙 QRB2210(Linux)과 STM32U585(MCU) 하이브리드 아키텍처를 가진 **아두이노 우노 Q(Arduino UNO Q)**에 퀀타이제이션된 YOLO 모델을 배포하여 실시간 제스처 기반 객체 추적 시스템을 구축하는 가이드라인입니다.

---

## 1. 하드웨어 구성 및 역할 정의 (Dual-Brain 구조)

아두이노 우노 Q는 고성능 리눅스 프로세서(MPU)와 실시간 제어용 MCU가 결합된 보드이므로 다음과 같이 역할을 분담합니다.

* **개발 환경 (Jetson Nano DevKit)**:
  - GPU 자원을 활용하여 손 제스처 및 사람 탐지용 YOLOv5 모델 학습을 수행합니다.
  - 최적화된 추적 로직 및 퀀타이제이션(ONNX / TFLite 변환)을 수행하는 호스트 역할을 합니다.
* **배포 환경 (Arduino UNO Q - MPU 측: Qualcomm QRB2210 / Linux)**:
  - 퀀타이제이션된 가벼운 YOLOv5n 모델(`.onnx` 또는 `.tflite`)을 ONNX Runtime / TensorFlow Lite 인터프리터 상에서 실행합니다.
  - 카메라 피드에서 사람과 제스처를 인식해 추적 목표의 각도/위치 편차(오차값)를 계산합니다.
  - 계산된 제어 명령을 보드 내부 시리얼 포트(예: `/dev/ttyHS0` 등)를 통해 MCU 측으로 전송합니다.
* **제어 환경 (Arduino UNO Q - MCU 측: STM32U585 / Arduino Sketch)**:
  - MPU로부터 수신한 시리얼 명령(속도, 방향, 각도 등)에 따라 서보모터/모터 실드를 실시간으로 제어합니다.

## 2. 전체 개발 및 배포 절차 (5단계 세부 로드맵)

### 🟢 [Phase 1] YOLOv5 커스텀 제스처 데이터셋 및 학습
YOLOv5 모델을 커스터마이징하여 손 제스처(주먹, 보자기)를 직접 인식하도록 데이터셋을 구축하고 모델을 학습시키는 단계입니다.
1. **1단계**: 주먹/보자기 제스처 이미지 수집 (클래스당 200~500장 이상, 다양한 각도/조명) 및 Roboflow/LabelImg 등을 이용한 바운딩 박스 라벨링 (YOLO format)
2. **2단계**: 학습/검증 데이터셋 폴더 구조 구성 및 `dataset.yaml` 설정 파일 생성
3. **3단계**: `yolov5n.pt` 또는 `yolov5s.pt` 사전 학습 모델을 기반으로 Jetson Nano/개발 서버에서 전이 학습(Transfer Learning) 실행

### 🟡 [Phase 2] YOLOv5 커스텀 모델 연동 및 젯슨 나노 최적화
학습 완료된 커스텀 가중치 모델(`best.pt`)을 로드하여 저전력/고효율 실시간 제스처 및 객체 추적 추론 루프를 개발하는 단계입니다.
1. **1단계**: 커스텀 `best.pt` 모델을 메모리 최적화 옵션(입력 320x320, FP16 반정밀도, `no_grad`)으로 로드하여 실시간 추론 스크립트 작성
2. **2단계**: 카메라 피드에서 검출된 제스처(주먹: 중지, 보자기: 추적 시작)에 따른 제어 로직 흐름 설계

### 🟠 [Phase 3] OpenCV KCF 트래커 결합 (하이브리드 추적)
YOLO의 지속적인 RAM/GPU 부하를 차단하고 실시간성을 확보하기 위해 경량 트래커를 연동하는 단계입니다.
1. **1단계**: OpenCV 내장 `KCF Tracker` 객체 생성 및 초기화 로직 구현
2. **2단계**: YOLO는 10프레임에 한 번만 실행하여 대상을 보정하고, 평소에는 KCF 트래커로 추적하는 하이브리드 루프 구현

### 🔵 [Phase 4] MPU - MCU 간 시리얼 통신 설계 (우노 Q 대비)
젯슨 나노(개발) 및 우노 Q 리눅스(MPU)에서 판별한 좌표를 아두이노 스케치(MCU)로 전송해 모터를 돌리는 단계입니다.
1. **1단계**: Python의 `pyserial`을 이용해 타겟의 화면 중심 오차 좌표(X, Y)를 시리얼 포트로 전송하는 코드 구현
2. **2단계**: 아두이노 스케치(C++)에서 이 값을 파싱해 서보모터를 움직여 카메라 각도를 타겟으로 일치시키는 모터 제어 코드 구현

### 🔴 [Phase 5] 아두이노 우노 Q 배포 및 최종 테스트
실제 하드웨어 타겟 보드에 업로드하여 동작시키는 마무리 단계입니다.
1. **1단계**: 학습 완료된 커스텀 가중치 모델(`.pt`)을 ONNX 포맷으로 변환하고 FP16/INT8로 양자화(Quantization) 진행
2. **2단계**: 아두이노 우노 Q의 데STM32U585 / Arduino Sketch)**:
비안 리눅스에 코드와 퀀타이제이션 모델을 포팅하고, 실제 주행/구동 테스트

## 3. 제스처 인식 기술 검토 및 최종 선정 결과

Jetson Nano(임베디드 보드)의 성능 한계와 실시간 구동성(30 FPS 이상)을 만족시키기 위해 기술 검토를 거쳐 다음과 같이 프로젝트 설계를 확정하였습니다.

* **선정 방식**: **공개 데이터셋(방법 B) 기반 YOLOv5n 1차 학습 후, 현장 오탐지 대응용 수동 촬영 데이터 파인튜닝(Fine-tuning) 하이브리드 전략**
  - **이유**: 처음부터 데이터를 모두 직접 촬영/라벨링하는 것(방법 A)은 구축 리소스가 과도하며, Google MediaPipe(방법 C)는 Jetson Nano 환경에서 라이브러리 컴파일 빌드 충돌 오류가 잦고, GPU 가속 활성화가 까다로워 CPU 연산 한계로 인한 프레임 저하(10~15 FPS) 및 CPU 자원 고갈 리스크가 큽니다.
  - **대응책**: Roboflow 등 오픈소스의 'Fist, Palm' 데이터셋을 활용해 빠르게 기본 인식 모델을 확보하고, 실제 구동 시 감지가 불안정한 특정 조명/구도의 이미지만 소량(50~100장) 추가 수집 및 어노테이션하여 파인튜닝(전이 학습)을 수행합니다. 이를 통해 개발 속도를 단축하면서도 30 FPS 이상의 초고속 추론 환경을 달성합니다.

## 4. 개발 시작 시 놓칠 수 있는 체크리스트

* **데이터셋 밸런스**: 주먹(Fist)과 보자기(Palm) 데이터의 장수가 불균형하면 한쪽만 잘 인식할 수 있으므로 가급적 클래스별 데이터 비율을 1:1로 유지합니다.
* **배경 다양성**: 학습 이미지는 손만 크게 찍힌 것 외에 실제 아두이노 우노 Q가 동작할 주변 환경(배경)이 포함된 이미지를 다수 확보해야 오탐지율이 내려갑니다.
* **UNO Q 내부 통신 채널**: 퀄컴 MPU와 STM32 MCU 간 통신에 사용되는 전용 내부 시리얼 포트 명칭(포트 디바이스명 및 보드레이트)을 매뉴얼에서 사전에 파악해 두어야 합니다.

## 5. 변경 이력 (Change Log)

* **2026-07-01**: 
  - OpenCV 기하학 기반 제스처 인식(Phase 1) 방식에서 YOLOv5 커스텀 모델 직접 학습 및 연동 방식으로 프로젝트 아키텍처 및 로드맵 변경 확정.
  - Google MediaPipe Hands와 YOLOv5 커스텀 학습 방안을 다각도로 비교 분석함. Jetson Nano에서의 빌드 이슈 및 GPU 가속 제약으로 인한 FPS 하락 우려로 MediaPipe 검토는 최종 보류함.
  - 공개 데이터셋(방법 B) 기반 학습 및 현장 오탐 대응용 촬영 데이터 추가 파인튜닝 하이브리드 전략을 최종 설계로 확정 및 기록.
* **2026-07-06**:
  - **모델 정확도 향상 및 TensorRT 교차 검증 전략 추가**:
    0.25배 이하의 초소형 모델로는 960장 이상의 제스처 데이터셋을 온전히 암기하지 못해 손 인식이 불안정해지는 Underfitting 문제 발견.
    이에 따라 모델 용량을 확장한 **0.50배 모델** 및 **1.0배 순정 모델(YOLOv5n)** 2종을 PC에서 학습시키기로 변경.
    두 모델을 모두 젯슨 나노에서 TensorRT FP16 가속 엔진(`.engine`)으로 컴파일하여 실시간 속도(30 FPS)와 인식 정확도의 최적 밸런스를 1대1 비교 검증한 후 배포 가중치를 결정하는 실험 전략 확정.
* **2026-07-23**:
  - **75% 슬림 신경망 다이어트 설계 및 UltraFast 113 FPS 달성 확정**:
    1.76M 파라미터(3.9MB) 슬림 신경망(`yolov5_gesture_slim.yaml`)을 완성하고, 젯슨 나노 4GB RAM 하드웨어에서 TensorRT FP16 가속 엔진 컴파일을 성공함.
    UltraFast (`320x240`) 모델에서 지연시간 **`8.6 ms` (0.008초)**, 실측 **`113.27 FPS`** 최고속 기록 달성 및 RAM 점유율 1.2GB 이하로 OOM 렉 0% 입증 완료.
  - **6-Class 확장 및 아두이노(Arduino) 시리얼 제어 이식 단계 진입**:
    `person`, `fist`, `palm`, `ok`, `like`, `no_gesture` 6개 클래스 지원 명세 확정 및 아두이노 시리얼 통신(115200 Baud) 제어 프로토콜 설계 착수.
